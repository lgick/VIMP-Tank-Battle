# Документация VIMP Tank Battle

Многопользовательская 2D онлайн-игра реального времени: авторитетный сервер (Node.js, Rapier 2D), клиент на PixiJS, обмен по WebSocket с бинарными снапшотами.

## Разделы

| Страница | О чём |
| --- | --- |
| [getting-started.md](getting-started.md) | Локальная настройка: установка, HTTPS-сертификаты, запуск, тесты, локальный мультиплеер |
| [architecture.md](architecture.md) | Общая архитектура: сервер/клиент/shared, игровой цикл, жизненный цикл соединения, ключевые инварианты |
| [gameplay.md](gameplay.md) | Игровой процесс: раунды, команды, статистика, голосования, чат-команды, управление, оружие, боты, кики |
| [server.md](server.md) | Серверные модули: VIMP-фасад, менеджеры, физика Rapier, мета-модули, инфраструктура, боты |
| [client.md](client.md) | Клиентские модули: MVC-компоненты, интерполяция, prediction, спавн снарядов, рендеринг, звук |
| [network.md](network.md) | Синхронизация сервер‑клиент: WebSocket-порты, бинарный snapshot-кадр (v3), форматы данных, RTT |
| [configuration.md](configuration.md) | Конфигурация: переменные `.env`, все файлы `src/config/`, данные `src/data/` |
| [extending.md](extending.md) | Расширение: новая карта, оружие, звук, клиентская сущность |
| [deployment.md](deployment.md) | Развертывание: подготовка VPS, добавление/удаление серверов, CI/CD |

## С чего начать

- **Хочу запустить локально** → [getting-started.md](getting-started.md)
- **Хочу понять, как всё устроено** → [architecture.md](architecture.md), затем [server.md](server.md) / [client.md](client.md) / [network.md](network.md)
- **Хочу поднять свой сервер** → [deployment.md](deployment.md)
- **Хочу добавить карту/оружие** → [extending.md](extending.md)

> Документация поддерживается вместе с кодом: при изменении функционала соответствующая страница обновляется в том же изменении (правило зафиксировано в [CLAUDE.md](../../CLAUDE.md)).
