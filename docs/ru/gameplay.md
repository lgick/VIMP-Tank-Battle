# Игровой процесс

Командный танковый deathmatch: две команды (`team1`, `team2`) уничтожают друг друга по раундам, третья «команда» — наблюдатели (`spectators`). Все правила авторитетны на сервере.

## Путь игрока

1. **Подключение и авторизация** — игрок вводит имя (и модель танка); имя проверяется валидатором и дедуплицируется сервером. При полном сервере (`maxPlayers`, люди + боты) — очередь с позицией на экране.
2. **Наблюдатель** — после загрузки карты игрок попадает в игру спектатором: видит мир, камера следует за наблюдаемым игроком, клавиши `n`/`p` переключают цель наблюдения. Сразу приходит окно выбора команды.
3. **Выбор команды** — через голосование-меню (`m` → Switch team). Если в команде нет свободных респаунов, сервер пытается вытеснить бота; иначе — отказ («Team ... is full»).
4. **Игра** — в начале раунда игрок получает танк на респауне. В первые `teamChangeGracePeriod` (10 с) раунда смена команды применяется немедленно; позже — игрок доигрывает раунд (или уходит в наблюдатели), смена вступает со следующего раунда.
5. **Смерть** — игрок становится наблюдателем до конца раунда, камера переключается на убийцу.

## Раунды и победа

- **Раунд** длится `roundTime` (по умолчанию 2 минуты). Победа — **team wipe**: уничтожение всех участников команды противника (боты считаются). Победившая команда получает командный фраг (`score +1` в шапке), проигравшая — поражение (`deaths +1`); всем проигрывается victory/defeat и показывается «{team} WINS!».
- Если время раунда истекло без победителя — новый раунд запускается без начисления очков.
- Между раундами пауза `roundRestartDelay` (5 с). В начале раунда мир пересоздаётся: все живы, панель — с дефолтными значениями, респауны раздаются по командам.
- **Карта** живёт `mapTime` (10 минут). По истечении сервер автоматически запускает голосование за следующую карту (`mapsInVote` вариантов); если никто не проголосовал — время текущей карты продлевается.

## Статистика (Tab)

Правила начисления ([RoundManager.reportKill](../../src/server/core/RoundManager.js)):

1. Уничтоженный игрок получает поражение (`deaths +1`) и статус `dead` до конца раунда.
2. Уничтоживший соперника получает фраг (`score +1`).
3. Уничтоживший игрока **своей** команды теряет фраг (`score −1`).
4. Самоубийство — только поражение (`deaths +1`), фраги не изменяются.
5. За team wipe команда-победитель получает `score +1`, проигравшая — `deaths +1` (итоги в шапке таблицы).
6. Столбец `latency` — текущий RTT игрока (у ботов пусто).

Изменения статистики рассылаются в момент появления; сортировка таблиц — на клиенте (`score` по убыванию, затем `deaths` по возрастанию).

## Голосования (клавиша `m`)

Система коллективных решений ([Vote](../../src/server/modules/Vote.js) + [VoteCoordinator](../../src/server/core/VoteCoordinator.js)):

- **Меню** — окно с пунктами «Switch team» и «Suggest map».
- **Запуск игроком или системой** — предложение карты игроком (если игрок в игре один — карта меняется сразу, без голосования), голосование за ботов, автоматический выбор карты по таймеру.
- **Очередь** — голосование, созданное во время активного, ставится в очередь и запускается после.
- **Кулдаун** — после голосования на тему действует блокировка `timeBlockedVote` (30 с) от спама.
- **Время жизни** — `voteTime` (10 с); окна с `timeOff: true` (меню) не закрываются по таймеру.
- **Пагинация** — списки длиннее 7 разбиваются на страницы (Back/More).
- **Ничья** — победитель выбирается случайно среди набравших максимум.

Формат обмена — в [network.md](network.md#голосование-порт-16).

## Чат (клавиша `c`) и команды

Обычный текст — сообщение команде/всем (длина ограничена сервером, 60 символов). Сообщения, начинающиеся с `/`, — команды ([CommandProcessor](../../src/server/core/CommandProcessor.js)):

| Команда | Действие |
| --- | --- |
| `/name <ник>` | Смена имени (с проверкой и системным сообщением) |
| `/timeleft` | Оставшееся время карты |
| `/mapname` | Название текущей карты |
| `/bot <N> [team]` | Создать N ботов (в команду или с равномерным распределением); `/bot 0 [team]` — удалить ботов |
| `/nr` | Новый раунд — **только в dev-режиме** |

`/bot` доступен только активным игрокам. Если активных людей больше одного, вместо немедленного исполнения запускается голосование; исполнение команды перезапускает раунд.

## Управление

Наборы клавиш переключает сервер по статусу (наблюдатель/игрок):

- **Наблюдатель**: `n` — следующий игрок, `p` — предыдущий.
- **Игрок**: `w/s` — газ/назад, `a/d` — поворот, `k/l` — поворот башни, `u` — башня в центр, `j` — огонь, `n/p` — следующее/предыдущее оружие.
- **Режимы** (в любом статусе): `c` — чат, `m` — голосование, `Tab` — статистика; `Esc`/`Enter` — управление внутри режимов.

Раскладка настраивается в `client.js` (`modules.controls`), команды и их типы — в `game.js` (`playerKeys`), см. [configuration.md](configuration.md#клавиши-spectatorkeys-playerkeys).

## Оружие и танк

У танка два оружия (переключение `n`/`p`, активное подсвечено на панели):

- **`w1` — пуля (hitscan)**: мгновенный луч, урон 40, дальность 1500, боезапас 200. Попадание считает сервер лучом; клиент рисует трассер немедленно.
- **`w2` — бомба (explosive)**: физический снаряд, закладывается и детонирует через таймер; урон 70 в эпицентре с падением по радиусу 50, боезапас 100.

Здоровье — 100. Состояние танка (`condition`) визуально деградирует с уроном (дым), при 0 танк уничтожен. Характеристики — в [configuration.md](configuration.md#srcdata--игровые-данные).

## Панель HUD

Слева направо: время раунда, здоровье, боезапас `w1`/`w2` (активное оружие подсвечивается). Наблюдателю значения скрыты (пустая панель). Значения сбрасываются к дефолтным каждый раунд.

## Боты

Серверный ИИ ([src/server/modules/bots/](../../src/server/modules/bots/)): полноценные участники — занимают слоты `maxPlayers`, попадают в статистику, водят танки и стреляют через тот же ввод, что игроки. Навигация — поиск пути по сетке карты + пространственная сетка для поиска целей. Добавляются `/bot` или голосованием; при заходе человека в полную команду бот может быть вытеснен.

## Кики

- **Бездействие**: игрок без ввода/чата дольше `idleKickTimeout.player` (2 мин) кикается (наблюдатели — нет, по умолчанию `null`).
- **Сеть**: RTT выше `maxLatency` (300 мс) или `maxMissedPings` (5) пропущенных пингов подряд — соединение закрывается с техническим сообщением.

## Карты

`pool mini`, `canopy`, `garden` — тайловые карты с респаунами по командам, статической геометрией и динамическими объектами (передаются в снапшоте). Смена — голосованием или по таймеру карты. Добавление новой — [extending.md](extending.md#новая-карта).
