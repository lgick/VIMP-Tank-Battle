## Синхронизация

| Объект                    | Метод                                               |
| :------------------------ | :-------------------------------------------------- |
| **Свой танк**             | Client Prediction + Reconciliation                  |
| **Чужие танки**           | Snapshot Interpolation                              |
| **Снаряды (визуал)**      | Client-side spawn (создаем сразу для отзывчивости)  |
| **Снаряды (физика/урон)** | Server Authoritative (просчет попаданий на сервере) |
| **Взрывы / Урон**         | Server-time events (события по времени сервера)     |

## Реализовация синхронизации

### 1. Для локального игрока (Свой танк)

**Подход:** `Client Prediction + Reconciliation`

- **Изоляция:** Отделяешь локальный танк от всех остальных сущностей (`entitiesOnCanvas`).
- **Локальная симуляция:** Считаешь движение локально в каждом кадре (например, в
  `requestAnimationFrame`) строго по нажатым клавишам.
- **Коррекция (Reconciliation):** Серверные данные используются не для движения, а для коррекции.
  Если локальная и серверная позиции расходятся, плавно подтягиваешь (`correct`) локальную позицию
  к серверной.
- **Результат:** Игрок управляет танком моментально, без инпут-лага. При этом сервер остаётся
  «истиной» и рассинхронизация исключена.

### 2. Для других танков (Чужие танки)

**Подход:** `Snapshot Interpolation`

- **Snapshot Buffer:** Хранишь последние 3–4 пакета от сервера с позициями и временными метками
  (`timestamp`).
- **Смещение времени:** В цикле `update` вычисляешь время отрисовки:
  `renderTime = serverTime - bufferTime` (обычно буфер составляет 100–150 мс). Это значит, что мы
  видим чужие танки немного в прошлом.
- **Интерполяция:** Находишь два снапшота (пакета) вокруг значения `renderTime` ➔ делаешь линейную
  интерполяцию (`lerp`) для координат `x`, `y` и угла `rotation`.
