### Философия гибридного подхода модели игрока в физическом мире RapierJS.

Идея в том, чтобы разделить "ощущения" от "правил":

1.  **Что мы берем от Симуляции (Impulses):**
    - **Ощущение Веса (Массы):** Мы продолжаем использовать `applyImpulse()` для разгона. Именно
      масса тела (`rigidBody.mass()`) в Rapier определяет, насколько "лениво" и "тяжело" танк будет
      отзываться на нажатие газа. Легкий танк рванет с места, тяжелый будет медленно набирать
      скорость.
    - **Инерция:** Когда танк врезается в другой объект, именно его масса и скорость (импульс)
      определяют, кто кого оттолкнет.
    - **Плавное торможение:** Мы используем `damping`, чтобы танк плавно останавливался, а не
      замирал на месте, как вкопанный.

2.  **Что мы берем от Аркады (Прямой контроль):**
    - **Ограничение Максимальной Скорости:** После того как движок всё честно посчитал, мы, как
      "боги" этого мира, вмешиваемся и говорим: "Так, быстрее 250 км/ч ехать нельзя". Мы
      принудительно срезаем скорость (`setLinvel`), не давая симуляции выйти из-под контроля.
    - **Контроль Заноса (`lateralGrip`):** Мы смотрим на результат симуляции и говорим: "Меня не
      волнует, что по законам физики танк должно занести. Гусеницы держат дорогу". И мы вручную
      гасим боковую скорость с помощью корректирующего импульса.
    - **Поворот:** Мы используем `applyTorqueImpulse`, чтобы танк поворачивался с предсказуемой и
      приятной скоростью, а не пытаемся симулировать сложное взаимодействие гусениц с поверхностью.

### Аналогия: Гоночный симулятор

Представьте гоночную игру. Движок симулирует массу машины, сцепление шин, аэродинамику. Это
**симуляция**. Но при этом в игре есть "помощники": антиблокировочная система (ABS), контроль тяги
(Traction Control) и она не дает машине улететь в стратосферу после аварии. Это **аркадные
правила**, наложенные поверх симуляции, чтобы в игру было весело и удобно играть.

**Танк — это та же гоночная машина.**

### Основные свойства игровой физики:  mass - масса inertia - инерция

**Velocity** - линейная скорость **Forces** - внешняя сила воздействия **Friction** - сопротивление
(трение) **Restitution** - упругость (отскок)

**Damping** - затухание **Gravity** - гравитация

### Как это выглядит в коде игрового цикла (Порядок имеет значение!)

Ваш `gameLoop` должен выполняться в строгой последовательности, чтобы гибридный подход работал
правильно:

```javascript
const MAX_TANK_SPEED = 250.0;
const LATERAL_FRICTION_FACTOR = 0.8; // 0.0 = дрифт, 1.0 = как по рельсам

function gameLoopTick(dt) {
  // Перебираем все активные танки
  for (const tank of allActiveTanks) {
    const rigidBody = tank.rigidBody;
    const keys = tank.keys; // Нажатые клавиши

    // --- Фаза 1: Намерение игрока (СИМУЛЯЦИЯ) ---
    // Игрок "давит на газ", прикладывая силу. Масса танка влияет на ускорение.
    if (keys.forward) {
      applyForwardImpulse(rigidBody, dt);
    }
    if (keys.left || keys.right) {
      applyTurningTorque(rigidBody, keys, dt);
    }

    // --- Фаза 2: Коррекция и "Правила Игры" (АРКАДА) ---
    // Мы НЕ ДАЕМ симуляции вести себя слишком реалистично.

    // Правило #1: Убираем нежелательный занос
    applyLateralFriction(rigidBody, LATERAL_FRICTION_FACTOR);

    // Правило #2: Ограничиваем максимальную скорость
    clampMaxSpeed(rigidBody, MAX_TANK_SPEED);
  }

  // --- Фаза 3: Шаг физического мира ---
  // Теперь Rapier берет ВСЕ приложенные импульсы и коррекции
  // и просчитывает конечный результат столкновений и перемещений.
  world.step();

  // --- Фаза 4: Синхронизация ---
  // Забираем финальные, скорректированные данные для отправки клиентам.
  syncPhysicsToECS();
}

// Вспомогательные функции, которые вы уже видели:
function applyLateralFriction(rigidBody, factor) {
  /* ... код гашения боковой скорости ... */
}
function clampMaxSpeed(rigidBody, maxSpeed) {
  const vel = rigidBody.linvel();
  const speed = Math.hypot(vel.x, vel.y);
  if (speed > maxSpeed) {
    const factor = maxSpeed / speed;
    rigidBody.setLinvel({ x: vel.x * factor, y: vel.y * factor }, true);
  }
}
```

### Итог в виде таблицы

| Параметр           | Чистая Симуляция (Плохо)                                           | Чистая Аркада (Плохо)                                       | **Наш Гибридный Подход (Отлично)**                                                     |
| :----------------- | :----------------------------------------------------------------- | :---------------------------------------------------------- | :------------------------------------------------------------------------------------- |
| **Вес (Масса)**    | Танк реалистично разгоняется, но его не остановить.                | Танк двигается как картонный, нет ощущения массы.           | **Танк ощущается тяжелым при разгоне, но слушается команд.**                           |
| **Макс. Скорость** | Неограниченная. Танк может улететь за карту и стать неуправляемым. | Скорость задана жестко, танк стартует и тормозит мгновенно. | **Плавный разгон до заданного, контролируемого предела.**                              |
| **Занос (Дрифт)**  | Сильный и неконтролируемый, как на льду. Играть неприятно.         | Танк едет как по рельсам, нет ощущения физики.              | **Легкий, управляемый "дрифт", который делает управление приятным, но предсказуемым.** |

Так что не бойтесь. Вы не отказываетесь от физики, вы её **"укрощаете"** для нужд геймплея. Это и
есть работа гейм-дизайнера и программиста физики в играх.
